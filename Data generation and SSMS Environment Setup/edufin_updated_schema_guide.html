🎯 Business Context

This dataset represents a comprehensive education loan portfolio management system for EduFin Credit Solutions, an Indian NBFC specializing in education financing. 🗺️ Dimension Tables

🗺️ DIM_STATE - State Master Dimension

Purpose: Master reference for all Indian states and union territories

Records: 5,000 | Business Key: state_name

Field Name Data Type Constraints Description
state_id INT PRIMARY KEY PK Unique identifier for state
state_name VARCHAR(30) UNIQUE, NOT NULL UQ Official state/UT name
region ENUM ('North','South','East','West','Central','Northeast') Geographic region classification

📋 Business Rules

  • Covers all 30 Indian states and union territories
  • Regional classification for economic analysis
  • Single source of truth for state data
  • Variations created for 5000 record requirement
🏙️ DIM_CITY - City Master Dimension

Purpose: Master reference for all cities with state relationships

Records: 5,000 | Business Key: city_name + state_id

Field Name Data Type Constraints Description
city_id INT PRIMARY KEY PK Unique identifier for city
city_name VARCHAR(50) UNIQUE, NOT NULL UQ City name (unique within state)
state_id INT FOREIGN KEY → dim_state FK Reference to parent state
tier_classification ENUM ('Tier1','Tier2','Tier3') City development tier

📋 Business Rules

  • Tier1: Metro cities (Mumbai, Delhi, Bangalore, Chennai, etc.)
  • Tier2: Major cities (Jaipur, Lucknow, Indore, etc.)
  • Tier3: Smaller cities and towns
  • City-state relationship enforced via foreign key
  • Enables geographic and economic analysis

👥 Core Business Tables

👥 CUSTOMERS - Customer Master Data

Purpose: Central repository of all loan applicants and customers

Records: 5,000 | Business Context: Individual borrowers

Field Name Data Type Constraints Description
customer_id INT PRIMARY KEY PK Unique customer identifier
full_name VARCHAR(100) NOT NULL Customer's complete name
phone_number VARCHAR(15) UNIQUE UQ Contact number (+91 format)
email_address VARCHAR(100) UNIQUE UQ Email for communication
city_id INT FOREIGN KEY → dim_city FK Customer's current city
current_address TEXT NOT NULL Complete postal address
annual_income DECIMAL(12,2) NOT NULL Annual income in ₹
cibil_score INT CHECK (300-900) Credit score (300-900 range)
employment_type VARCHAR(30) NOT NULL Employment category
employer_name VARCHAR(100) Current employer
date_of_birth DATE NOT NULL Birth date
gender VARCHAR(10) CHECK ('Male','Female') Gender
education_level VARCHAR(30) Highest qualification

💼 Business Characteristics

  • Age Range: 18-30 years (typical education loan applicants)
  • Income Distribution: Varies by city tier and employment type
  • CIBIL Distribution: Government employees (650-850), Private (600-800)
  • Employment Mix: 45% Private, 20% Government, 20% Self-employed, 15% Others
🏫 INSTITUTIONS - Educational Institution Master

Purpose: Repository of all educational institutions where loans are disbursed

Records: 5,000 | Business Context: Universities, Colleges, Institutes

Field Name Data Type Constraints Description
institution_id INT PRIMARY KEY PK Unique institution identifier
institution_name VARCHAR(150) NOT NULL Complete institution name

  • Tier3 Institutions: ₹50K-2.5L fees, 35-70% placement, 500-6K students
  • Accreditation: NAAC A+/A/B+, NBA, UGC recognition levels

💰 Transaction Tables

💰 LOANS - Core Loan Transactions

Purpose: Central transaction table linking customers to educational institutions

Records: 5,000 | Business Context: Education loan disbursements

Field Name Data Type Constraints Description
loan_id INT PRIMARY KEY PK Unique loan identifier
customer_id INT FOREIGN KEY → customers FK Borrower reference
institution_id INT FOREIGN KEY → institutions FK Educational institution
loan_amount DECIMAL(12,2) NOT NULL, CHECK (>0) Principal amount in ₹
loan_status VARCHAR(20) NOT NULL Active/Closed/Defaulted/Overdue
interest_rate DECIMAL(5,2) CHECK (5.0-20.0) Annual interest rate %
loan_tenure_months INT CHECK (12-120) Repayment period in months
application_date DATE NOT NULL Loan application date
disbursement_date DATE NOT NULL Loan disbursement date
maturity_date DATE NOT NULL Loan maturity date
emi_amount DECIMAL(10,2) NOT NULL Monthly EMI in ₹
purpose_of_loan VARCHAR(50) Course Fees/Living/Equipment

💼 Loan Characteristics

  • Amount Range: ₹1.5L - ₹10L+ (course fees + living expenses)
  • Interest Rates: 9-17% based on CIBIL score and employment
  • Tenure: 3-8 years typical for education loans
  • Status Distribution: ~65% Active, ~20% Closed, ~12% Defaulted, ~3% Overdue
  • Processing Time: 10-45 days from application to disbursement
💳 PAYMENTS - EMI Payment Transactions

Purpose: Detailed record of all EMI payments made against loans

Records: 5,000 | Business Context: Monthly installment payments

Field Name Data Type Constraints Description
payment_id INT PRIMARY KEY PK Unique payment identifier
loan_id INT FOREIGN KEY → loans FK Associated loan
payment_date DATE NOT NULL Transaction date
payment_amount DECIMAL(10,2) NOT NULL, CHECK (>0) Amount paid in ₹
payment_method VARCHAR(30) NOT NULL UPI/Net Banking/Card/Cheque
payment_status VARCHAR(20) NOT NULL Success/Failed/Pending
late_fee DECIMAL(8,2) DEFAULT 0 Late payment charges
principal_component DECIMAL(10,2) DEFAULT 0 Principal portion of payment
interest_component DECIMAL(10,2) DEFAULT 0 Interest portion of payment
outstanding_balance DECIMAL(12,2) NOT NULL Remaining loan balance

💼 Payment Characteristics

  • Success Rate: 92% of payments are successful
  • Method Distribution: UPI (35%), Net Banking (25%), Cards (25%), Others (15%)
  • Late Fees: Applied to 8% of payments (₹500-₹2000 range)
  • Partial Payments: 5% of payments are partial (0.4-0.8x EMI)

Purpose: Comprehensive tracking of defaulted loans and collection activities

Records: 5,000 | Business Context: Risk management and recovery

Field Name Data Type Constraints Description

💼 Collection Characteristics

  • Status Distribution: Active (45%), Legal Action (30%), Written Off (15%), Settled (10%)
  • Recovery Rates: Settled (40-90%), Legal Action (10-60%), Active (0-50%)
  • Timeline: Legal action typically after 180+ days overdue
  • Agent Workload: 10 collection agents handling portfolio

🔗 Relationship Summary

Core Business Relationships

  • dim_state (1) → dim_city (M) - States contain multiple cities
  • dim_city (1) → customers (M) - Cities have multiple customers
  • dim_city (1) → institutions (M) - Cities have multiple institutions
  • customers (1) → loans (M) - Customers can have multiple loans
  • institutions (1) → loans (M) - Institutions serve multiple students
  • loans (1) → payments (M) - Each loan has multiple payment records
  • customers (1) → defaults_collections (M) - Customers can have multiple defaults
  • loans (0..1) → defaults_collections (1) - Each loan may have one default record

Dimension Relationships

  • dim_city (1) → geographic_demographics (M) - City demographic data
  • dim_state (1) → economic_indicators (M) - State economic data over time

🎯 Relationship Benefits

  • Zero Redundancy: City and state names stored once in dimension tables
  • Referential Integrity: All foreign keys properly constrained
  • Query Performance: Optimized joins through proper indexing
  • Data Consistency: Master data managed centrally
  • Scalability: Easy to add new cities/states without duplication

Data Integrity Constraints

Constraint Type Implementation Business Impact
Primary Keys All tables have surrogate integer keys Ensures unique record identification
Foreign Keys Proper referential integrity enforced Prevents orphaned records
Unique Constraints Phone, email, city_name, state_name Prevents duplicates in master data
Check Constraints CIBIL scores (300-900), rates, percentages Ensures realistic data ranges
NOT NULL Essential fields cannot be empty Guarantees completeness for critical data
ENUM Types Tier classification, regions, statuses Standardizes categorical data

Business Logic Rules

🎯 Key Business Rules Implemented

  • Date Consistency: application_date ≤ disbursement_date ≤ maturity_date
  • Age Restrictions: Customers aged 18-30 years (education loan demographic)
  • CIBIL Impact: Interest rates inversely correlated with CIBIL scores
  • Tier Economics: Tier1 cities have higher incomes, fees, and living costs
  • Employment Patterns: Government employees get lower interest rates
  • Default Logic: Self-employed customers have higher default probability
  • Recovery Rates: Vary by collection status and customer profile
  • Payment Methods: UPI dominance reflects modern Indian payment trends

📋 Usage Guidelines & Best Practices

Recommended Query Patterns

🎯 Efficient Join Patterns

Geographic Analysis: Always join through dimension tables

customers → dim_city → dim_state → economic_indicators

Portfolio Analysis: Use customer-loan-payment chain

customers → loans → payments

Risk Analysis: Combine customer, loan, and default data

customers → loans → defaults_collections

Index Recommendations

Table Recommended Indexes Use Case
customers city_id, cibil_score, employment_type Geographic and risk analysis
loans customer_id, institution_id, loan_status, disbursement_date Portfolio analysis and reporting
payments loan_id, payment_date, payment_status Payment tracking and analysis
defaults_collections customer_id, loan_id, collection_status, default_date Collection management
economic_indicators state_id, quarter Time series economic analysis

Data Loading Guidelines

⚠️ Load Order for Referential Integrity

  1. dim_state - Master state data first
  2. dim_city - Cities with state references
  3. customers, institutions - Master entities with city references
  4. loans - Transactions linking customers and institutions
  5. payments, defaults_collections - Transaction details
  6. geographic_demographics, economic_indicators - Analytics data
